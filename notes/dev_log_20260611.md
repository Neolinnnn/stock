## 1. 行動清單空白（觀望等待更明確訊號）— 根治

**症狀**：行動清單卡全顯示「觀望等待更明確訊號」，無買入訊號與評分。

**根因鏈**：CI 環境有其他套件提供 `indicators` 命名空間，shadow 本地 `indicators/` 目錄
→ `from indicators.stock_analyzer import analyze_stock` 失敗（`No module named`）
→ `_ANALYZER_OK=False` → `trend_analysis` 從未寫入 qualified → UI fallback。

**修法（唯一有效）**：`importlib.util.spec_from_file_location` 直接從絕對路徑載入，
完全跳過 sys.path。實作在 [scripts/build_docs.py](../scripts/build_docs.py) `_load_analyze_stock()`。
**sys.path 系列修法全部無效**（insert root、insert cwd 都被 shadow）。

**本機快速排查**（不用看 CI log）：
```bash
python3 -c "import sys; sys.path.insert(0,'scripts'); import build_docs; print(build_docs._ANALYZER_OK)"
# 預期 True；CI log 搜「分析引擎載入成功」確認
```

## 3. 進場策略實驗室（第一訊號源的加強層）

**腳本**：[scripts/backtest_entry_lab.py](../scripts/backtest_entry_lab.py)（重跑秒回，資料快取 `backtest_cache/`）
**樣本**：128 筆雙篩選 qualified 訊號（19 檔、2025/04~2026/06），進場=次日開盤，無前視偏差。

**結論**：「**量比≥1.2 或 MACD柱>0且增長**」（動能確認）最優：
| | 基準 | 動能確認 |
|---|---|---|
| 勝率 | 66% | 77%（Wilson 下界 65%）|
| 平均報酬 | +8.4% | +12.4% |
| PF | 2.52 | 4.12 |
- walk-forward 前 75% / 後 79%，無衰減
- 搭配追蹤停損15%：平均 +31%、PF 8.1
- **反直覺**：「強勢族群」59%、「cv_sharpe≥1.0」62%——都**低於基準**（族群已強=個股已過熱）

**整合**：`build_docs.py` 給每檔 qualified 加 `entry_strategy`（vol_ratio / macd_momentum / grade A/B/C），
行動清單卡顯示「⚡ A雙動能 / B單動能 / C動能不足」徽章。

## 4. 領頭羊突破系統（第二訊號源，bottom-up）

**理念**：雙篩選是 top-down（族群→個股），抓輪動波但訊號晚；
突破系統不看族群，直接抓「自己先動」的領頭羊，與雙篩選互補。

**條件**：大盤>MA60（開關）→ 收盤創20日高 → 量比≥1.5 → 法人5日淨買超>0
→ >MA20、MA20上揚、乖離≤10%。同股不重複進場。

**回測**：[scripts/backtest_breakout_lab.py](../scripts/backtest_breakout_lab.py)，84 檔全追蹤池、183 筆：
- TP18/SL15：勝率 65%、+7.0%、PF 2.19
- 追蹤停損15%：勝率 55%、**+15.1%、PF 3.22**（持有 42 天）
- walk-forward 前 48% → 後 64%（改善中）
- 結果存 `docs/breakout_lab.json`

**每日掃描**：[scripts/breakout_scan.py](../scripts/breakout_scan.py)
- 技術條件用 `price_cache/` 免 API；只對通過者查籌碼（省額度）
- 與回測邏輯驗證 6/6 一致；輸出 `docs/breakout.json`
- CI 步驟「Breakout scan」在 daily scan 之後
- UI：今日掃描「🐏 領頭羊突破」金框卡區（無訊號也顯示狀態；歷史日期不顯示）

## 6.5 過期 BUY bug + 買轉弱觀察區（6/12）

**Bug：強力賣出的智邦出現在行動清單**
- 根因鏈：6/9 MA5 微幅黃金交叉（差 3 點）產生 BUY → 6/10 隨即死叉，
  但 SELL 需 RSI5 > 35（防殺低設計），當日 RSI5 = 34.9 → **SELL 被超賣抑制**
  → `current_signal` 取「近 5 根最後一個訊號」殘留舊 BUY → CV 達標照樣入選
- **設計教訓：跌得越兇（RSI 越超賣），SELL 越不觸發，過期 BUY 越安全**
- 修法（`batch_scan.py::analyze_stock`）：BUY 必須當下仍為多頭結構
  （MA5 > MA20），否則降為 HOLD。只動當前訊號判定，回測用的
  `generate_signals` 不動
- qualified 真正的入選條件（`daily_scan.py` ~880 行）：
  `signal=='BUY'` + cv_sharpe≥0.3 + cv_win_rate≥0.4 + cv_max_dd≤0.2

**買轉弱觀察區（由此 bug 衍生的功能）**
- `build_docs.py::build_signal_flips()`：回看 5 個交易日 qualified，
  今日掉出清單者進入觀察區，payload 欄位 `signalFlips`
- 分類邏輯：
  - **回檔測試**＝仍站 MA20 且距訊號價跌 < 8% → 二次進場候選，
    卡片顯示動能回穩條件（量比≥1.2 / MACD 翻強，沿用實驗室標準），
    達標標綠「可評估再進場」
  - **轉弱**＝破 MA20 或跌 > 8% → 警示「反彈至 MA20 為減碼點」
- UI 在行動清單下方，與 action-card 同款（黃框=回檔測試、紅框=轉弱）
- 隱性價值：**訊號品質回饋面板**——轉弱卡常態多於行動清單
  ＝雙篩選在當前市況失效的早期警報

  OpenBB 不能取代；國際指標 macro 模組**早已用 yfinance 實作**（market_regime.py）

  macro/requirements.txt 與主 requirements.txt 都有，CI 兩者都裝）
- 驗證：2330 收盤價與 FinMind 5/5 完全一致；8086 上櫃自動 fallback `.TWO` 成功
- 注意：yfinance 需 `auto_adjust=False` 拿原始收盤價（與 FinMind 一致）
